# MCP ToolLatch v0.3.0-beta.1

Release date: 2026-06-05

## Positioning

`v0.3.0-beta.1` is a beta release for small-scope external trials and developer-local validation. MCP ToolLatch is a local policy, approval, and audit gateway for MCP tool calls. It is not a production-grade sandbox.

## New In This Release

- `scan --deep` safely starts configured stdio MCP servers and calls `initialize` plus `tools/list`.
- Policy profiles: `observe`, `balanced`, and `strict`.
- `doctor` diagnostics for scan, policy, audit, and setup status.
- `apply` dry-run/write flow, `--yes` write alias, `claude` client alias, and `restore` backup recovery.
- `config paths` and `rules list` inspection commands.
- Redacted audit JSONL querying and `logs export` for JSON/CSV.
- Real process-level proxy E2E coverage with a fake MCP server fixture.
- MCP Inspector validation guide and fixture-only local demo.

## Verified Scenarios

- Source install, typecheck, tests, build, and lint.
- Dist CLI smoke for help, version, scan, deep scan, doctor, init, policy check, wrap config, logs, rules, and config paths.
- Local tarball pack, fresh install, and `npx toollatch` smoke.
- Proxy E2E for `initialize`, `tools/list`, safe file read, `.env` block, `secret.pem` block, and dangerous shell command block.
- Apply/restore dry-run, write, backup, idempotency, and corrupted-config protection for Cursor, Claude Desktop alias, and VS Code style configs.
- `scan --deep` fault cases: slow server, bad JSON, stderr noise, and abnormal exit.
- MCP Inspector Web launch and Inspector CLI method calls through ToolLatch.

## Install And Try

From source:

```bash
pnpm install
pnpm build
node packages/cli/dist/index.js --version
```

Local tarball trial:

```bash
pnpm --dir packages/cli pack --pack-destination ./tmp/packs
mkdir -p ./tmp/install-smoke
cd ./tmp/install-smoke
npm init -y
npm install ../packs/mcp-toollatch-cli-0.3.0-beta.1.tgz
npx toollatch scan --json
```

## Known Limits

- Not a production-grade sandbox.
- No kernel-level isolation.
- No guarantee against all prompt injection or malicious MCP server behavior.
- No Web Dashboard, RBAC, cloud policy center, or full sandbox.
- Deep scan does not call tools and does not prove a server is safe.
- Balanced profile denies non-interactive shell confirmations by default.

## Next Steps
