# v0.4.0-beta.1 Draft Release Notes

Status: draft, not released.

## Highlights

- Domain allow/deny policy for URL arguments and curl/wget-style shell commands.
- Explicit safe shell allowlist with dangerous command deny precedence.
- Standardized rule explanations in policy decisions, proxy block responses, and audit logs.
- Audit JSONL rotation, multi-file reads, redacted JSON/CSV/Markdown export.
- `scan --server` filtering and stronger suspicious MCP tool metadata detection.
- Doctor issues now include category, severity, suggested command, and doc link.
- Apply/write summaries now include impact, backup preview, and rollback command.
- GitHub Actions quality workflow and CLI pack/fresh-install smoke.
- Phase 3 docs for policy reference, threat model, Inspector checks, trial flow, and feedback triage.

## Verified Scope

- Local CLI and stdio proxy only.
- Temporary fixtures and fake MCP servers only for automated compatibility tests.
- No real secrets, tokens, private keys, certificates, or production logs required.

## Deferred

- Web Dashboard, RBAC, cloud policy center, SaaS, and complete sandbox.
- npm publish, git tag, and hosted Release creation.
- Real token-backed GitHub MCP validation.
- Custom local rule pack, policy migration command, logs summary/report, and DB risk fixture.

## Recommended Release Gate

Before publishing this beta, run:

```bash
pnpm install
pnpm typecheck
pnpm test
pnpm build
pnpm lint
```

Then run CLI smoke, pack smoke, fresh install smoke, sensitive-file checks, MCP Inspector validation, and the Phase 3 acceptance audit checklist.
